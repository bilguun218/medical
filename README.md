# NOVYTAS Corporate Website

Premium corporate medical website and database-driven CMS for NOVYTAS LLC.

The public site supports Mongolian and English routes, while the admin dashboard is available directly at `/admin`. The public UI does not expose an admin link.

## Stack

- Next.js 15 App Router
- React 19 and TypeScript
- Tailwind CSS with reusable UI primitives
- Framer Motion for lightweight transform/opacity animation
- Prisma ORM with PostgreSQL
- Auth.js credentials login
- Zod and React Hook Form
- TipTap rich text editor
- pdfmake for inquiry PDF export

## Main Features

- Premium corporate medical UI with responsive public pages
- Database-driven CMS content for home, about, contact, header, footer, and SEO
- Admin dashboard content editing
- Media upload and URL storage
- Product catalog with category filtering
- Products use Prisma autogenerated `id` values only, no slug field
- News management with rich text, publish/unpublish, edit, and delete
- Contact and product quote requests
- Admin inquiry detail pages
- Inquiry export as Word, Excel, and PDF
- SEO records for editable public pages

## Local Setup

1. Install dependencies:

```bash
npm install
```

2. Create `.env` from `.env.example` and set:

```env
DATABASE_URL=
AUTH_SECRET=
ADMIN_EMAIL=
ADMIN_PASSWORD=
NEXT_PUBLIC_SITE_URL=
BLOB_READ_WRITE_TOKEN=
```

3. Run database migrations:

```bash
npm run prisma:dev
```

4. Seed initial data:

```bash
npm run seed
```

5. Start development:

```bash
npm run dev
```

Open:

```txt
http://localhost:3000/mn
http://localhost:3000/admin
```

## Database Notes

Products and product categories no longer use slugs. Product detail pages and APIs use:

```prisma
id String @id @default(cuid())
```

Forms and requests do not require administrators to enter IDs manually.

## CMS Content

Editable CMS content is stored in `SiteSetting` as structured JSON and validated with Zod. Public pages read CMS values dynamically, so saved admin changes appear on the website immediately.

CMS-managed areas:

- Home page
- About page
- Contact page
- Header
- Footer
- SEO

## Media

Uploaded media metadata is stored in PostgreSQL.

Storage behavior:

- Local development without Blob credentials saves files under `public/uploads`.
- Production/Vercel with Vercel Blob credentials saves files to Vercel Blob and stores the Blob URL.

Supported media types:

- Images
- Logos
- Banners
- PDF/documents for product attachments

## Inquiries

Contact and product quote requests are stored as `ContactInquiry` records. Admins can:

- View all requests
- Open request detail pages
- Update status
- Delete requests
- Export requests as Word, Excel, or PDF

## Deployment

For Vercel:

1. Add all required environment variables.
2. Use PostgreSQL for `DATABASE_URL`.
3. Create/connect a Vercel Blob store for production media uploads.
4. Ensure migrations are deployed:

```bash
npm run prisma:migrate
```

Vercel Blob setup:

1. In Vercel, open the project Storage tab.
2. Create a Blob store or connect an existing Blob store to the project.
3. Ensure Vercel adds Blob credentials to the selected environments. `BLOB_READ_WRITE_TOKEN` works, while connected Vercel projects can also use `BLOB_STORE_ID` with Vercel OIDC.
4. Pull env values locally when needed:

```bash
vercel env pull
```

If media upload fails on Vercel with an error like `ENOENT: no such file or directory, mkdir '/var/task/public...'`, Blob is not connected/configured for that deployment. Vercel serverless functions cannot persist uploads to `public/uploads`; configure Vercel Blob and redeploy.

4. Build command:

```bash
npm run build
```

The latest local verification completed successfully with:

```bash
npm run typecheck
npm run build
```
